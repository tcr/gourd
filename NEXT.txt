 Key Decisions

 - Pipeline Separation: gourd-codegen handles proc-macro interface (go! macro +
   #[verify_rust_output] attribute), while gourd-codegen-core houses all
   transpilation logic and validation for reuse/testing.
 - Validation Strategy: Uses real go build and cargo check invocations in temp
   directories at compile time for accurate, immediate error feedback.
 - Parsing Approach: Relies on syn custom Parse implementations for high-level
   Go AST nodes, with low-level parsing using manual token advancement for
   Go-specific syntax.
 - Slice Literal Parsing Fix: Replaced syn::bracketed! macro with manual
   advance_to and parse::<proc_macro2::TokenTree>() loops to correctly handle
   Go's separate [ and ] punctuation tokens, avoiding syn's bracket group
   expectations.
